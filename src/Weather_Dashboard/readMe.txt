

Project: Weather Dashboard

    Description:
        
        Develop a weather dashboard that allows users to search for the current weather
        and a 5-day forecast for any city. This project will require you to integrate
        with a weather API, handle asynchronous data fetching, and dynamically update
        the UI based on user input.

    Features:

        Search for City:

        Allow users to input a city name and search for the current weather.
        Display a list of matching cities if the input is ambiguous.

    Current Weather Display:

        Show the current weather for the selected city, including temperature,
        humidity, wind speed, and weather conditions.
    
    5-Day Forecast:

        Displa2y a 5-day weather forecast for the selected city.
        Include information such as daily temperature, weather conditions,
        and icons representing the weather.

    Favorites:

        Allow users to save their favorite cities.
        Display weather information for favorite cities on the dashboard.

    Responsive Design:

        Ensure the application is fully responsive and works well on both
        desktop and mobile devices.

    Error Handling:

        Handle errors gracefully, such as invalid city names or network issues.

    Steps to Build:
        
        Setup:

            Create the project structure using HTML, CSS, and JavaScript.
            Set up a basic HTML layout and CSS styling for the weather dashboard.

        API Integration:

            Sign up for a free API key from a weather service like OpenWeatherMap.
            Write JavaScript functions to fetch weather data from the API based on user input.

        Search Functionality:

            Implement a search bar where users can input a city name.
            Display a list of matching cities and allow users to select the correct one.

        Display Weather Data:

            Create functions to display current weather and the 5-day forecast.
            Use JavaScript to dynamically update the DOM with the fetched data.

        Favorites:

            Implement functionality to save favorite cities to local storage.
            Display weather information for favorite cities on the dashboard.

        Responsive Design:

            Use CSS to ensure the dashboard is responsive.
            Test the application on different devices and screen sizes.

        Error Handling:

            Add error handling for invalid city names and network issues.
            Display user-friendly error messages.
